## 一、原计划整体评估与改进说明

### ✅ 原计划优点
1. **技术栈递进合理**：Python基础 → Web开发 → 数据库 → LLM理论 → Agent核心 → RAG → 项目实战，符合学习曲线
2. **劳逸结合设计好**：工作日6h + 周六缓冲4h + 周日全休，避免 burnout
3. **理论实践结合**：每天上午看视频 + 下午动手写代码，符合编程学习规律
4. **求职导向明确**：最后一周专门安排简历和面试，目标清晰

### ⚠️ 原计划主要问题与改进点

| 问题 | 改进方案 |

| **缺少Git学习安排**（工程化板块提到了但没排） | 第二周新增Git专项学习，代码管理是开发基本功 |

- **重点对比：** 你已经会Java OOP，重点记Python和Java的区别：
  - `self` 不是 `this`（必须显式写在参数里）
  - 没有 `private` 关键字（靠下划线约定）

  - 鸭子类型（不关心类型，只关心有没有这个方法）
- **为什么学：** 大型项目必须用OOP组织代码，LangChain里全是类和继承

#### Day 22（8.05 周二）- Transformer + Tokenization
**视频1：** [李沐 Transformer论文精读](https://www.bilibili.com/video/BV1pu411o7BE)（1小时，必看）
**视频2：** [3Blue1Brown Transformer动画（中配）](https://www.bilibili.com/video/BV182sYz4EZf/)（20分钟，直观理解Q/K/V）
**视频3：** 搜「大模型Token是什么」看10分钟科普
- **学习目标：** 能用自己的话讲清楚Transformer的编码器-解码器结构，理解自注意力机制
- **为什么学：**
  - 所有大模型的底层都是Transformer，不懂原理就是调包侠
  - 理解Token才能搞懂为什么有长度限制、为什么按Token收费

#### Day 23（8.06 周三）- Prompt Engineering 提示词工程
**视频：** [吴恩达 ChatGPT Prompt Engineering（中文字幕）](https://www.bilibili.com/video/BV1No4y1t7Zn)（约1.5小时，业界最权威）
- **学习目标：** 掌握角色设定、Few-shot、CoT思维链、格式约束等核心技巧
- **为什么学：**
  - 同样的模型，好的Prompt和差的Prompt效果天差地别
  - Agent的System Prompt写得好不好直接决定智能体的行为
  - 这是成本最低、见效最快的优化手段
- **下午练手：**
  - 用你的DeepSeekClient，写5个不同类型的Prompt做对比实验
  - 测试维度：角色设定 / Few-shot示例 / CoT思维链 / JSON格式控制

#### Day 24（8.07 周四）- Function Calling 原理与实现 🔑
**视频：** 搜「Function Calling 原理 大模型」看30-40分钟
- **学习目标：** 理解Tool Schema定义，掌握完整的「LLM决策→执行工具→结果回传」流程
- **为什么学：**
  - Function Calling是Agent的基石，没有它LLM只能聊天不能干活
  - 让大模型能调用计算器、查数据库、搜网络，能力边界瞬间打开
  - Agent开发工程师最核心的知识点，没有之一
- **下午练手：**
  - 手写两个简单工具：`get_current_time()` 和 `calculate(expression)`
  - 定义Tool Schema，让LLM在对话中自动决定要不要调用工具
  - 仔细观察：LLM什么时候选择直接回答，什么时候选择调用工具

#### Day 25（8.08 周五）- Agent 核心：ReAct 循环 🔑
**视频1：** 搜「AI Agent 开发入门 Python」看1小时实战
**视频2：** 搜「ReAct Agent 原理」看20-30分钟
- **学习目标：** 理解Think → Act → Observe 循环，能手写一个最简Agent
- **为什么学：**
  - ReAct是目前最主流的Agent范式，LangChain的Agent默认就是ReAct
  - 理解了ReAct循环，你就理解了Agent的本质：推理 + 行动交替
  - 面试必问：「Agent的核心循环是什么？」
- **下午练手（周项目启动）：**
  - **AI刷题助手（最简版）**
  - 实现ReAct循环 + 三个工具：
    - `search_question(知识点)` 从题库搜题
    - `generate_question(知识点, 难度)` 让LLM出新题
    - `check_answer(题目, 用户答案)` 批改答案

#### Day 30（8.13 周三）- RAG 全流程 🔑
**视频：** [2026版RAG企业级项目实战](https://www.bilibili.com/video/BV137dUBhEQL/) 看核心流程部分
- **学习目标：** 跑通RAG五步流程：加载 → 切分 → 向量化 → 存储 → 检索
- **为什么学：**
  - RAG是目前大模型落地最成熟的方案，解决幻觉和知识过时问题
  - 企业知识库、客服机器人、AI助教，本质都是RAG
  - Agent岗位90%的项目都会涉及RAG，重中之重
- **RAG五步详解：**
  1. **加载 Load**：用PyPDFLoader读PDF，TextLoader读txt
  2. **切分 Split**：RecursiveCharacterTextSplitter把长文档切成块
  3. **向量化 Embed**：用Embedding模型把文本变成向量
  4. **存储 Store**：存入ChromaDB等向量数据库
  5. **检索 Retrieve**：用户提问 → 转向量 → 相似度匹配 → 返回相关片段
- **下午练手：**
  - 找一份离散数学笔记或PDF当素材
  - 跑通 Load → Split → Embed → Store → Retrieve 全流程
  - 用 `retriever.invoke("欧拉回路是什么")` 测试检索效果好不好

## 附录 C：卡住了怎么办

1. **同一个视频看不懂** → 换一个up主的，不同人讲法不同，B站搜关键词按播放量排
2. **代码报错看不懂** → 把完整报错丢给DeepSeek问「这个报错什么意思，怎么解决」
3. **卡了超过2天** → 跳过，缓冲日集中回来补。不要让一个知识点拖死整个进度
4. **太累不想学** → 当天直接休息。42天缺1-2天完全不影响，硬扛效率更低
5. **某个板块完全学不懂** → 标记下来，先往后走，很多东西学着学着就通了
6. **Agent调不通** → 先把每一步的输入输出都print出来，看看到底哪一步出问题了
