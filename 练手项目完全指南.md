### 项目1：BMI计算器 + 成绩等级转换（Day1）

#### 功能要求
**BMI计算器：**
- 用户输入身高（米）和体重（公斤）
- 计算 BMI = 体重 / 身高²
- 根据中国标准判断体型：
  - < 18.5 → 偏瘦
  - 18.5 ~ 23.9 → 正常
  - 24 ~ 27.9 → 偏胖
  - ≥ 28 → 肥胖

**成绩等级转换：**
- 输入 0-100 的分数
- 输出对应等级：
  - 90-100 → A
  - 80-89 → B
  - 70-79 → C
  - 60-69 → D
  - < 60 → F

#### 实现提示
- 用 `input()` 获取用户输入，注意转成 float/int
- 用 `if-elif-else` 做分支判断
- 输入非法（比如输入字母）可以先不处理，后面学了异常处理再加

#### 验收标准
- 输入身高1.75、体重65，输出BMI≈21.22，正常
- 输入59分输出F，输入60分输出D，输入90分输出A

#### 参考代码骨架
```python
# BMI计算器
height = float(input("请输入身高（米）："))
weight = float(input("请输入体重（公斤）："))
bmi = weight / (height ** 2)

if bmi < 18.5:
    category = "偏瘦"
elif bmi < 24:
    category = "正常"
elif bmi < 28:
    category = "偏胖"
else:
    category = "肥胖"

print(f"你的BMI是：{bmi:.2f}，体型：{category}")
```

```python
        break
```

```python
    elif cmd[0] == "list":

        if 0 <= idx < len(todos):
            removed = todos.pop(idx)
            print(f"已删除：{removed}")
```

```python
    words = text.split()
```

### 项目6：Student类 + 继承（Day8）

#### 功能要求
**Student 基类：**
- 属性：姓名、学号、成绩列表
- 方法：

  - `average()` 返回平均分

**GraduateStudent 子类：**
- 继承 Student
- 新增属性：导师姓名
- 重写 ranking() 方法，研究生标准更严格（85分以上才算优）

#### 实现提示
- 类定义 `class Student:`，初始化用 `__init__(self, name, student_id)`
- 继承 `class GraduateStudent(Student):`
- 子类调用父类方法用 `super().__init__(...)`
- `@property` 装饰器可以把方法变成属性调用，试试把 average 改成属性

#### 验收标准
- 学生添加3门成绩，average返回正确平均值
- 研究生84分是良不是优，体现了标准不同

#### 参考代码骨架
```python
class Student:
    def __init__(self, name, student_id):
        self.name = name
        self.student_id = student_id
        self.scores = []
    
    def add_score(self, score):
        self.scores.append(score)
    
    @property
    def average(self):
        if not self.scores:
            return 0
        return sum(self.scores) / len(self.scores)
    
    def ranking(self):
        avg = self.average
        if avg >= 90:
            return "优"
        elif avg >= 80:
            return "良"
        elif avg >= 70:
            return "中"
        elif avg >= 60:
            return "及格"
        else:
            return "不及格"


class GraduateStudent(Student):
    def __init__(self, name, student_id, supervisor):
        super().__init__(name, student_id)
        self.supervisor = supervisor
    
    def ranking(self):
        avg = self.average
        if avg >= 85:  # 研究生标准更严
            return "优"
        elif avg >= 75:
            return "良"
        elif avg >= 65:
            return "中"
        elif avg >= 60:
            return "及格"
        else:
            return "不及格"
```

### 项目7：DeepSeekClient 封装（Day10）⭐ 重点项目

#### 功能要求
封装一个 DeepSeekClient 类，包含：
- `__init__(api_key, base_url, model)` 初始化，从环境变量读key
- `chat(messages)` 非流式对话，返回完整回复字符串
- `chat_stream(messages)` 流式对话，逐字打印
- 统一的异常处理：网络错误、超时、API返回错误码

#### 实现提示
- 环境变量：`import os` + `from dotenv import load_dotenv`，key存在 `.env` 文件里
- HTTP请求：用 `requests.post()`
- 请求体格式参考 DeepSeek 官方文档：
  ```json
  {
    "model": "deepseek-chat",
    "messages": [{"role": "user", "content": "你好"}]
  }
  ```
- 流式：`stream=True`，迭代 `response.iter_lines()`
- 异常捕获：`try-except` 捕获 requests.exceptions 下的各种异常

#### 验收标准
- 传入普通messages能正常得到回复
- API Key填错时，程序不崩溃，打印友好的错误提示
- 流式输出能一个字一个字地打印出来
- `.env` 文件不提交到Git（记得加 .gitignore）

#### 参考代码骨架
```python
import os
import requests
from dotenv import load_dotenv

load_dotenv()  # 加载 .env 文件中的环境变量

class DeepSeekClient:
    def __init__(self):
        self.api_key = os.getenv("DEEPSEEK_API_KEY")
        self.base_url = "https://api.deepseek.com/v1/chat/completions"
        self.model = "deepseek-chat"
        self.headers = {
            "Authorization": f"Bearer {self.api_key}",
            "Content-Type": "application/json"
        }
    
    def chat(self, messages):
        """非流式对话"""
        payload = {
            "model": self.model,
            "messages": messages,
            "stream": False,
            "temperature": 0.7
        }
        try:
            response = requests.post(
                self.base_url,
                headers=self.headers,
                json=payload,
                timeout=30
            )
            response.raise_for_status()  # 4xx/5xx 抛出异常
            data = response.json()
            return data["choices"][0]["message"]["content"]
        except requests.exceptions.Timeout:
            return "错误：请求超时，请稍后重试"
        except requests.exceptions.RequestException as e:
            return f"请求出错：{str(e)}"
    
    def chat_stream(self, messages):
        """流式对话，逐字yield"""
        payload = {
            "model": self.model,
            "messages": messages,
            "stream": True
        }
        try:
            response = requests.post(
                self.base_url,
                headers=self.headers,
                json=payload,
                stream=True,
                timeout=30
            )
            response.raise_for_status()
            for line in response.iter_lines():
                if line:
                    line_str = line.decode('utf-8')
                    if line_str.startswith("data: "):
                        data_str = line_str[6:]
                        if data_str == "[DONE]":
                            break
                        import json
                        data = json.loads(data_str)
                        delta = data["choices"][0].get("delta", {}).get("content", "")
                        if delta:
                            yield delta
        except requests.exceptions.RequestException as e:
            yield f"请求出错：{str(e)}"


# 使用示例
if __name__ == "__main__":
    client = DeepSeekClient()
    
    # 非流式
    messages = [{"role": "user", "content": "用一句话解释什么是Agent"}]
    result = client.chat(messages)
    print(result)
    
    # 流式
    print("\n--- 流式输出 ---")
    for char in client.chat_stream(messages):
        print(char, end="", flush=True)
```

#### 参考代码骨架
```python
from pydantic import BaseModel, Field, field_validator
from typing import List

class Message(BaseModel):
    role: str  # user / assistant / system
    content: str
    
    @field_validator('role')
    @classmethod
    def check_role(cls, v):
        if v not in ['user', 'assistant', 'system']:
            raise ValueError(f'role必须是user/assistant/system，不能是{v}')
        return v

class ChatRequest(BaseModel):
    model: str = "deepseek-chat"
    messages: List[Message]
    temperature: float = Field(default=0.7, ge=0, le=2, description="温度值0-2")


# 测试：正确数据
valid_req = ChatRequest(
    messages=[Message(role="user", content="你好")],
    temperature=0.7
)
print("校验通过：", valid_req.model_dump())

# 测试：非法数据（会报错）
try:
    invalid_req = ChatRequest(
        messages=[{"role": "invalid_role", "content": "测试"}],
        temperature=3.0
    )
```

### 项目9：题库 API（FastAPI CRUD）（Day15-16）

#### 功能要求
一个题库管理的 RESTful API，包含：
- `POST /questions` 新增题目

- `DELETE /questions/{id}` 删除题目

题目字段：id、题干、选项（列表）、正确答案、难度、类别

#### 实现提示
- 先存内存列表里，不用数据库
- 路径参数：`@app.get("/questions/{question_id}")`
- 查询参数：`category: str = None, difficulty: str = None`
- 请求体：定义一个 Question 类（Pydantic BaseModel）
- 自动生成文档：访问 `/docs` 就是 Swagger UI

#### 验收标准
- Swagger文档里能测试所有5个接口
- 添加3道题，列表能查到
- 按难度筛选只返回对应难度的题
- 随机抽题每次可能不一样

#### 参考代码骨架
```python
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel
from typing import List, Optional
import random

app = FastAPI(title="题库API")

# 数据模型
class QuestionCreate(BaseModel):
    question_text: str
    options: List[str]
    correct_answer: str
    difficulty: str  # easy / medium / hard
    category: str

class Question(QuestionCreate):
    id: int

# 内存存储
questions_db = []
next_id = 1

# 新增题目
@app.post("/questions", response_model=Question)
def create_question(q: QuestionCreate):
    global next_id
    new_q = Question(id=next_id, **q.model_dump())
    questions_db.append(new_q)
    next_id += 1
    return new_q

# 获取列表（支持筛选+分页）
@app.get("/questions", response_model=List[Question])
def list_questions(
    category: Optional[str] = None,
    difficulty: Optional[str] = None,
    skip: int = 0,
    limit: int = 10
):
    result = questions_db
    if category:
        result = [q for q in result if q.category == category]
    if difficulty:
        result = [q for q in result if q.difficulty == difficulty]
    return result[skip : skip + limit]

# 获取单题
@app.get("/questions/{question_id}", response_model=Question)
def get_question(question_id: int):
    for q in questions_db:
        if q.id == question_id:
            return q
    raise HTTPException(status_code=404, detail="题目不存在")

# 随机抽题
@app.get("/questions/random", response_model=Question)
def random_question():
    if not questions_db:
        raise HTTPException(status_code=404, detail="题库为空")
    return random.choice(questions_db)

# 更新
@app.put("/questions/{question_id}", response_model=Question)
def update_question(question_id: int, q: QuestionCreate):
    for i, item in enumerate(questions_db):
        if item.id == question_id:
            updated = Question(id=question_id, **q.model_dump())
            questions_db[i] = updated
            return updated
    raise HTTPException(status_code=404, detail="题目不存在")

# 删除
@app.delete("/questions/{question_id}")
def delete_question(question_id: int):
    for i, q in enumerate(questions_db):
        if q.id == question_id:
            questions_db.pop(i)
            return {"message": "删除成功"}
    raise HTTPException(status_code=404, detail="题目不存在")
```

- Swagger测试全部通过

#### 参考代码骨架

```python
from sqlalchemy import create_engine, Column, Integer, String
from sqlalchemy.ext.declarative import declarative_base

SQLALCHEMY_DATABASE_URL = "sqlite:///./quiz.db"
```

### 项目13：Function Calling 双工具（Day24）

#### 功能要求
实现两个工具，让LLM自动决定调用哪个：
1. `get_current_time()` → 返回当前时间字符串
2. `calculate(expression: str)` → 计算数学表达式结果

完整流程：用户提问 → LLM判断是否需要工具 → 调用工具 → 把结果塞回去 → LLM生成最终回答

#### 实现提示
- 工具定义成JSON Schema格式，传给DeepSeek的 tools 参数
- LLM返回时如果有 `tool_calls` 字段，说明要调用工具
- 解析函数名和参数，执行对应函数
- 把工具结果以 `role: tool` 的消息加回messages，再调一次LLM得到最终回答

#### 验收标准
- 问「现在几点」→ 自动调用get_current_time，返回准确时间
- 问「1234乘以5678等于多少」→ 自动调用calculate，返回正确结果
- 问「你好」→ 不调用工具，直接回答

#### 参考代码骨架
```python
import json
from datetime import datetime
from deepseek_client import DeepSeekClient  # 用你之前封装的

client = DeepSeekClient()

# 工具定义（Schema）
tools = [
    {
        "type": "function",
        "function": {
            "name": "get_current_time",
            "description": "获取当前时间",
            "parameters": {"type": "object", "properties": {}, "required": []}
        }
    },
    {
        "type": "function",
        "function": {
            "name": "calculate",
            "description": "计算数学表达式，支持加减乘除",
            "parameters": {
                "type": "object",
                "properties": {
                    "expression": {"type": "string", "description": "数学表达式，如'3+5*2'"}
                },
                "required": ["expression"]
            }
        }
    }
]

# 工具实现
def get_current_time():
    return datetime.now().strftime("%Y-%m-%d %H:%M:%S")

def calculate(expression):
    try:
        # 简单实现，生产环境要用更安全的方式
        result = eval(expression)
        return str(result)
    except:
        return "计算错误"

# 执行工具调用
def execute_tool(name, arguments):
    args = json.loads(arguments)
    if name == "get_current_time":
        return get_current_time()
    elif name == "calculate":
        return calculate(args["expression"])
    return "未知工具"

# 带工具调用的完整对话
def chat_with_tools(user_input):
    messages = [{"role": "user", "content": user_input}]
    
    # 第一次调用LLM，可能返回工具调用
    response = client.chat_with_tools(messages, tools)  # 需要在client里加一个支持tools的方法
    
    # 如果有工具调用
    if response.get("tool_calls"):
        for tool_call in response["tool_calls"]:
            func_name = tool_call["function"]["name"]
            func_args = tool_call["function"]["arguments"]
            tool_result = execute_tool(func_name, func_args)
            
            messages.append(response)  # 把assistant的tool_call消息加回去
            messages.append({
                "role": "tool",
                "tool_call_id": tool_call["id"],
                "name": func_name,
                "content": tool_result
            })
        
        # 第二次调用LLM，拿到最终回答
        final_response = client.chat(messages)
        return final_response
    else:
        return response["content"]
```

### 项目14：AI刷题助手（ReAct Agent）（Day25-26）⭐ 核心项目

#### 功能要求
实现一个最简版 ReAct Agent，三个工具：
1. `search_question(keyword)` → 从题库搜索相关题目
2. `generate_question(topic, difficulty)` → 让LLM生成新题
3. `check_answer(question, user_answer)` → 批改答案并给出解析

Agent 循环：思考 → 行动 → 观察 → 思考... → 给出最终答案

#### 实现提示
- ReAct的核心是System Prompt里告诉LLM：你要按 Thought/Action/Observation 的格式输出
- 解析 LLM 输出中的 Action 部分，执行对应工具
- 把 Observation（工具返回结果）拼回去，继续循环
- 设置最大循环次数，防止死循环
- 当LLM输出 Final Answer 时结束循环

#### 验收标准
- 用户说「给我出一道Python基础题」→ Agent调用generate_question出题
- 用户回答后 → Agent调用check_answer批改
- 用户问「有没有关于循环的题目？」→ Agent调用search_question搜题
- 能看到完整的思考过程（Thought/Action/Observation）

#### 参考代码骨架（极简ReAct实现）
```python
import json
from deepseek_client import DeepSeekClient

client = DeepSeekClient()

SYSTEM_PROMPT = """你是一个AI刷题助手。你可以使用工具来帮助用户练习题目。

请严格按照以下格式思考和行动：
Thought: 你现在的思考，分析用户需求
Action: 工具名(参数)
Observation: 工具返回的结果
...（可以多轮循环）
Thought: 现在我可以给出最终答案了
Final Answer: 你的最终回答

可用工具：
1. search_question(keyword) - 从题库搜索相关题目
2. generate_question(topic, difficulty) - 生成指定主题和难度的题目，difficulty可选easy/medium/hard
3. check_answer(question, user_answer) - 批改用户答案，给出解析

注意：
- 每次只调用一个工具
- 不需要工具时直接给出Final Answer
- 最多思考5轮"""

# 工具实现（简化版）
def search_question(keyword):
    return f"搜索到关于'{keyword}'的3道题：\n1. ...\n2. ...\n3. ..."

def generate_question(topic, difficulty="medium"):
    # 实际项目中可以再调一次LLM生成，这里简化
    return f"【{difficulty}难度】{topic}相关题目：\n题目：Python中list和tuple的区别是什么？\n答案：..."

def check_answer(question, user_answer):
    return f"批改结果：\n你的回答：{user_answer}\n正确答案：...\n解析：..."

def execute_action(action_str):
    """解析 Action: func(args) 格式并执行"""
    import re
    match = re.match(r'(\w+)\((.+)\)', action_str.strip())
    if not match:
        return "无法解析Action格式"
    
    func_name = match.group(1)
    args_str = match.group(2)
    
    # 简单解析参数，实际项目可用更严谨的方式
    if func_name == "search_question":
        return search_question(args_str.strip("'\""))
    elif func_name == "generate_question":
        args = [a.strip("'\" ") for a in args_str.split(",")]
        return generate_question(*args)
    elif func_name == "check_answer":
        args = [a.strip("'\" ") for a in args_str.split(",", 1)]
        return check_answer(*args)
    return f"未知工具：{func_name}"

def react_agent(user_input, max_rounds=5):
    messages = [
        {"role": "system", "content": SYSTEM_PROMPT},
        {"role": "user", "content": user_input}
    ]
    
    for i in range(max_rounds):
        response = client.chat(messages)
        messages.append({"role": "assistant", "content": response})
        print(f"\n--- 第{i+1}轮 ---\n{response}\n")
        
        # 判断是否有最终答案
        if "Final Answer:" in response:
            return response.split("Final Answer:")[-1].strip()
        
        # 提取Action
        if "Action:" in response:
            action_part = response.split("Action:")[-1].split("\n")[0].strip()
            observation = execute_action(action_part)
            messages.append({
                "role": "user",
                "content": f"Observation: {observation}"
            })
        else:
            break
    
    return response


# 使用
if __name__ == "__main__":
    result = react_agent("给我出一道简单的Python基础题")
    print("\n最终回答：", result)
```

### 项目18：Redis 缓存 + 限流（Day32）

#### 功能要求
给AI助教加上生产级基础能力：
1. **Redis缓存**：相同问题1小时内直接返回缓存结果，不用重复调LLM
2. **Rate Limiting**：每个IP每分钟最多10次请求，超出返回429

#### 实现提示
- Docker启动Redis：`docker run -d -p 6379:6379 redis:7-alpine`
- Python客户端：`pip install redis`
- 缓存key：对问题做hash，`cache:question:{hash}`，设置EX=3600
- 限流：`rate_limit:ip:{ip}` 计数器，INCR + EXPIRE 组合实现固定窗口
- FastAPI可以用依赖注入实现限流中间件

#### 验收标准
- 连续问两次相同问题，第二次更快（走缓存）
- 用脚本快速发11个请求，第11个返回429
- 1分钟后又能正常请求

#### 参考代码骨架
```python
import redis
import hashlib
from fastapi import Request, HTTPException

r = redis.Redis(host="localhost", port=6379, decode_responses=True)

# 缓存
def get_cached_answer(question: str):
    key = f"cache:qa:{hashlib.md5(question.encode()).hexdigest()}"
    return r.get(key)

def set_cached_answer(question: str, answer: str, ttl=3600):
    key = f"cache:qa:{hashlib.md5(question.encode()).hexdigest()}"
    r.setex(key, ttl, answer)

# 限流
def rate_limit(request: Request, limit: int = 10, window: int = 60):
    ip = request.client.host
    key = f"rate_limit:{ip}"
    count = r.incr(key)
    if count == 1:
        r.expire(key, window)  # 第一次设置过期时间
    if count > limit:
        raise HTTPException(status_code=429, detail="请求过于频繁，请稍后再试")
    return True
```

## 通用调试技巧

### 代码跑不通时按这个顺序查
1. **看报错信息**：最后一行是错误类型，往上翻找到你的代码行号
2. **print调试**：每个关键变量都print出来，看是不是你预期的值
3. **缩小范围**：注释掉大部分代码，一点点加，定位到哪行出问题
4. **搜报错**：把报错信息复制去搜，90%的问题别人都遇到过
5. **问AI**：把完整代码+完整报错丢给DeepSeek，描述清楚你想做什么

### 写项目的推荐顺序
1. 先写最小可运行版本（能跑起来就行）
2. 再加功能，一个功能一个功能加
3. 每加完一个就测试，确认没问题再继续
4. 最后再整理代码结构、加注释
